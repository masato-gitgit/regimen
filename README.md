# OncoScheduler — がん化学療法レジメン管理システム

血液内科領域（多発性骨髄腫・リンパ腫等）の二重特異性抗体を含む
化学療法レジメンの投与スケジュール管理・投与前安全確認を支援する、
**単一施設・院内利用向けのローカル完結型 Web アプリ**です。

## ⚠️ 免責・利用上の注意

- 本ツールは投与計画の**参考情報**を提供するものであり、投与可否・用量の最終判断は必ず医療従事者が添付文書・院内プロトコルに基づいて行ってください。
- データはブラウザの `localStorage` にのみ保存されます。**ブラウザのデータ消去で全データが失われる**ため、定期的なバックアップ出力が必須です。
- 患者氏名等の個人情報を扱う場合は、施設の情報管理規程に従ってください（共有端末での利用時は患者IDのみでの運用を推奨）。

## 主な機能

- **患者管理**: 基本情報・BSA(DuBois式)・eGFR(日本人推算式)の自動計算、検査値記録
- **レジメンマスタ**: サイクル日数・薬剤・適用サイクル/Day の定義
- **特殊プロトコル対応**: テクベイリ / ルンスミオ(SC・IV) / タービー のステップアップ投与、休薬後の再漸増プロトコル、投与間隔延長
- **投与スケジュール**: 月間カレンダー、延期・スライド、8サイクル後の効果評価分岐
- **安全チェックリスト**: 投与前ダブルチェック（患者照合・検査値・曝露対策等）
- **薬剤必要量集計**: 1週間先まで
- **バックアップ/復元**: JSON エクスポート・インポート（バリデーション付き）

## データ保存仕様（localStorage）

| キー | 内容 |
|---|---|
| `onco_patients` | 患者配列（schedule 含む） |
| `onco_regimens` | レジメンマスタ配列 |
| `onco_drugs` | 薬剤マスタ配列 |
| `onco_migration_version` | データマイグレーションのバージョン番号 |
| `onco_last_backup_time` | 最終バックアップ日時（ISO文字列） |

データ構造を変更する場合は `src/utils/migrations.js` にマイグレーションを追加し、バージョン番号を上げてください（起動時に自動実行・冪等）。

## ディレクトリ構成

- `src/components/` … 画面コンポーネント群
  - `patient/` … 患者管理関連の分割コンポーネント（リスト、カレンダー、モーダル等）
- `src/utils/` … 純粋ロジック（投与量計算・eGFR・スケジュール生成・祝日・バリデーション・マイグレーション・プロトコル処理）※ テスト対象
- `src/hooks/` … useConfirm / useToast などのカスタムフック

## 既知の制限・今後の予定

- 複数端末間のデータ同期なし（単一端末運用前提）
- スケジュール生成・再開プロトコルのテスト拡充
- TreatmentCalendar / GlobalCalendar のカレンダー描画ロジックの共通化 (Phase 3 予定)
- RegimenList への Protocol Type 選択ドロップダウン追加 (UI改善予定)
